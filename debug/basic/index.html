Basic Vector Tile Rendering

This demo shows how to use the main renderTiles method on the BasicRenderer.

When testing locally, you can configure the defaults for the following text fields by modifying the defaults.js file (changes are ignored by git).

Style

The renderer lets you update individual bits of the style without fully reloading, but in this demo you can only give the full style and press "Apply" to create a new instance of the renderer with this applied.

tilesSpec

Here you need to specify an array of tiles that you want to render, for each tile you need to state the name of the source (as defined in the "style" above) as well as the tile's (z,x,y) values. Then you need to state where the tile is to be rendered on our imaginary infinite canvas (left,top,size). There is no particular origin defined for left and top values, they are really just relative values across all the tiles in this array and the values in drawSpec (below).

drawSpec

Here you need to specify which section of the imaginary infinite canvas you want to copy to the real canvas (destLeft, destTop,srcLeft,srcTop,width,height). As stated above, note that there is no particular origin in the imaginary infinite canvas, coordiantes are just relative to each other.
