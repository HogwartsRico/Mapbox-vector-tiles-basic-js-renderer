Basic Vector Tile Rendering

Introduction

This demo shows how to use the main renderTiles method on the BasicRenderer. This left hand pane has three text fields which expect JSON input (there is not much in the way of input validation, so be nice). On the right, the top section shows the "imaginary canvas", with all its source tiles and the region which is going to be "copied" from. And the bottom section on the right shows the "real output canvas" that the renderTiles method is actually trying to write to. Here the output canvas size is fixed at 1024x1024, but the imaginary canvas will grow as needed, based on your json specifications below. Don't forget to hit the "apply changes" button to update things.

When testing locally, you can configure the defaults for the following text fields by modifying the defaults.js file (use the command git update-index --skip-worktree debug/basic/defaults.js to ensure git ignores any changes you make).

Style

The renderer lets you update individual bits of the style without fully reloading, but in this demo you can only give the full style and press "Apply" to create a new instance of the renderer with this applied.

tilesSpec

Here you need to specify an array of tiles that you want to render, for each tile you need to state the name of the source (as defined in the "style" above) as well as the tile's (z,x,y) values. Then you need to state where the tile is to be rendered on our imaginary infinite canvas (left,top,size). There is no particular origin defined for left and top values, they are really just relative values across all the tiles in this array and the values in drawSpec (below). Note that the order of tiles here is irrelevant to the proper render, but you may need to reorder them to see the debug info better on the right.

drawSpec

Here you need to specify which section of the imaginary infinite canvas you want to copy to the real canvas (destLeft, destTop,srcLeft,srcTop,width,height). As stated above, note that there is no particular origin in the imaginary infinite canvas, coordiantes are just relative to each other. In this demo you can supply an array of drawSpecs, though the renderTiles function itself only accepts one at a time. You can also supply {... clear: true} if you want the renderTiles function to take responsibility for clearing anything before doing the render (it will do so at the last possible moment so that you shouldn't notice any flickering).
